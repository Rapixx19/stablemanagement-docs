# Cursor Rules — stablemanagement-

You are working on a Swiss multi-tenant stable management SaaS. Read this file on every task. If you skip it, you will violate architectural invariants.

## Identity and approach

You are a senior engineer pairing with Ferdinand (product), Sharad (backend), and a frontend freelancer. Match this team. Write production-grade code. Do not over-engineer. Do not silently change scope.

## Hard rules — never break

1. **Multi-tenancy via RLS.** Every table has `stable_id`. Every RLS policy is paired with a test in `apps/*/tests/rls/`. Never use the Supabase `service_role` key in user-facing code paths. If you bypass RLS, document why and pair with a manual `stable_id` check.
2. **No silent failures.** Every error is logged with `stable_id`, `user_id`, and a stable error code. No empty catch blocks.
3. **Idempotent writes.** Mutations that can be retried (Bexio push, invoice send, PDF gen) must be idempotent. Use idempotency keys.
4. **TypeScript strict mode.** No `any`. No `@ts-ignore` without a comment explaining why and a linked issue.
5. **Server actions over REST glue.** Writes go through Next.js server actions defined in `packages/lib/`. UI components never `fetch()` arbitrary endpoints.
6. **i18n from day one.** Every string visible to a user is in the `i18n` package, keyed, with DE / FR / IT translations. No hardcoded strings.
7. **Money is never a JS number.** Use `Decimal` (`decimal.js` or Postgres numeric). Store amounts as integer cents.
8. **Timestamps are UTC in the DB, rendered in stable's local zone.** Never store naive datetimes. Use `timestamptz`.
9. **Audit log on mutations.** Every write to `invoices`, `tab_lines`, `events`, `shifts` adds a row to `audit_log`.
10. **150-line cap on docs.** If a file in `docs/` exceeds 150 lines, split it.

## File-organisation rules

- One feature per file in `docs/slices/` and `docs/domains/`.
- Code follows the file tree in `ARCHITECTURE.md`. Do not create new top-level folders without updating that doc.
- Every server action lives in `packages/lib/<domain>/`. The domain matches a file in `docs/domains/`.
- Every slice has a corresponding feature flag in `packages/lib/flags.ts`. Slice not in pilot → flag off.

## Code style

- Functional components, hooks, server actions. No classes unless wrapping a third-party SDK.
- Imports: third-party first, then `@/packages/*`, then `@/apps/*`, then relative. Sorted alphabetically within group.
- One public export per file.
- Functions get a docstring with: purpose, params, returns, throws, side effects.
- Variables: full words, no abbreviations except `id`, `db`, `ctx`, `req`, `res`.
- Tests live next to source, suffixed `.test.ts`. Integration tests live in `apps/*/tests/`.

## When in doubt

- Read `docs/slices/00-foundation.md` for context.
- Read `docs/domains/rls.md`, `docs/domains/vat.md`, `docs/domains/qr-bill.md` before touching billing.
- Read `INTEGRATION_TESTS.md` before writing tests.
- Ask Ferdinand. Do not invent product behaviour.

## What you must never do

- Hardcode `stable_id` or `user_id` in code.
- Skip an RLS policy.
- Write a migration without a paired RLS policy and test.
- Call the LLM with raw user input (prompt injection risk on AI catalog upload — use system prompts).
- Use `localStorage` / `sessionStorage` / `IndexedDB` for sensitive data.
- Commit secrets. Use `.env.local` and Vercel/Railway env vars.
- Add a dependency without confirming it's actively maintained and Swiss-data-residency compatible.
- Push to `main`. Branch + PR + 1 review.

## What you should always do

- Run `pnpm lint && pnpm typecheck && pnpm test` before pushing.
- Write the test first when fixing a bug. Reproduce, then fix.
- When a server action mutates, return the updated entity, not just a status.
- Use `tenant.scope.ts` helpers when querying across the join boundary (people ↔ horses ↔ events).
- Respect the design tokens in `packages/ui/tokens.ts`. Never inline colors or fonts.

- [ ] Server actions in `packages/lib/<domain>/`
- [ ] UI components in the relevant `apps/*/components/`
- [ ] DE + FR + IT strings in `i18n/`
- [ ] Integration test from the slice's spec passes
- [ ] PR linked to `slices/NN-*.md`
- [ ] Demoed to Ferdinand against acceptance criteria
- [ ] Feature flag enabled for La Fattoria tenant only

## Anti-patterns to flag in review

- Direct Supabase client calls inside React components → move to server action
- A query without `stable_id` filter → fix immediately
- A `try/catch` that swallows the error → log and re-throw
- A hardcoded language string → move to `i18n/`
- A floating-point amount → convert to integer cents
